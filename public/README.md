# Peak Summit Capital --- Full Stack Architecture (FastAPI + Redis)

This branch extends the portfolio dashboard into a full-stack system
with a FastAPI backend and Redis caching layer.

It demonstrates backend engineering concepts including API design,
caching strategy, TTL expiration, and latency optimization.

------------------------------------------------------------------------

## System Architecture

React Frontend ↓ FastAPI Backend ↓ Redis (in-memory cache) ↓ Disk JSON
data (generated by Python)

This follows a cache-aside architecture pattern.

------------------------------------------------------------------------

## Why Redis Was Added

Originally, the frontend consumed static JSON files directly.

This branch introduces:

-   A REST API layer (FastAPI)
-   Redis-based in-memory caching
-   TTL-based expiration strategy
-   Measured latency improvements

------------------------------------------------------------------------

## Performance Improvements

Measured using curl:

Initial request (disk read): \~15 ms

Subsequent requests (Redis cache hit): \~1.5 ms

≈ 90% reduction in API response latency.

This eliminates repeated disk reads and reduces backend I/O overhead.

------------------------------------------------------------------------

## Caching Strategy

Endpoints:

-   /metrics
-   /performance/{range_key}

Flow:

1.  API checks Redis for cached key.
2.  If found → return cached JSON.
3.  If not found → load JSON from disk.
4.  Store in Redis with TTL.
5.  Return response.

TTL ensures data freshness while reducing recomputation.

------------------------------------------------------------------------

## Data Generation Pipeline

Data is generated via:

``` bash
python scripts/build_data.py
```

Key features of build_data.py:

-   Batched Yahoo Finance downloads
-   Exponential backoff retry strategy
-   Parquet price caching
-   Portfolio share reconstruction
-   Risk decomposition (systematic vs unsystematic)
-   Beta computation
-   Max drawdown calculation
-   YTD and since-inception metrics
-   Graceful fallback if API fails

If downloads fail but previous outputs exist, the system preserves
last-known-good data.

------------------------------------------------------------------------

## Running the Backend

Start Redis locally:

``` bash
redis-server
```

Start FastAPI:

``` bash
uvicorn app.main:app --reload
```

Test endpoint:

``` bash
curl http://localhost:8000/metrics
```
